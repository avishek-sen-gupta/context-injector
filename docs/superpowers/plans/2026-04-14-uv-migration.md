### Task 3: Update CI Workflow

**Files:**
- Modify: `.github/workflows/ci.yml`

- [ ] **Step 1: Replace setup-python with setup-uv**

Replace the entire `ci.yml` content with:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v5

      - uses: astral-sh/setup-uv@v6

      - name: Install dependencies
        run: uv sync --dev

      - name: Install semgrep
        run: uv tool install semgrep

      - name: Run tests
        run: uv run pytest tests/ -v
```

Key changes:
- `actions/setup-python@v6` → `astral-sh/setup-uv@v6` (uv handles Python installation via `.python-version`)
- `pip install ...` → `uv sync --dev` (installs from lockfile)
- Added `uv tool install semgrep` (isolated CLI tool, not a project dep)
- `python -m pytest` → `uv run pytest` (runs inside the managed venv)

## Development Setup

This project uses [uv](https://docs.astral.sh/uv/) for dependency management during development.

```bash
# Install uv (if not already installed)
curl -LsSf https://astral.sh/uv/install.sh | sh

# Install all dependencies (runtime + dev) into .venv/
uv sync --dev

# Run tests
uv run pytest tests/ -v
```

The development environment (`.venv/` managed by uv) is independent of the hook deployment. Contributors use `uv sync --dev` for local testing; end users run `install-governor.sh` which checks that runtime deps are available in the system Python.
```
